1

Initialize Project

Clone the repo. Then run the ./initialize.sh script to automatically provision MySQL databases and deploy a Staging and Production environment to Heroku.

2

Build and Test

Develop in true Test-Driven fashion with the Cypress.io test runner alongside your code. Start your app in test mode with hot reloading with the simple 'yarn dev' command.

3

CI/CD Pipeline

Configure a few environment variables and you are ready to send your project through CircleCI. It will run tests locally, and if they pass, deploy and test against staging. Fiinally, with manual approval, deploy and test production.

4

Relax and Repeat

You'll sleep soundly knowing your app is covered. Set minimum code coverage thresholds for Pull Requests. Test artifacts like reports, videos, and screenshots are all stored as CI artifacts for easy insight. Passing the pipeline means you are ready to move on to the next feature!
